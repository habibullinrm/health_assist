# Структура проекта Health Assist

Проект представляет собой систему цифрового ассистента здоровья, состоящую из нескольких микросервисов.

## Основные компоненты

### 1. `main-app` (Backend)
Основной сервер приложения, реализованный на **FastAPI**.
- **`app/`**: Основная логика приложения.
    - **`api/`**: Маршрутизаторы (routers) и эндпоинты API.
    - **`core/`**: Базовые настройки, конфигурация, безопасность.
    - **`crud/`**: Операции работы с базой данных (Create, Read, Update, Delete).
    - **`models/`**: SQLAlchemy модели базы данных.
    - **`schemas/`**: Pydantic схемы для валидации данных.
    - **`services/`**: Бизнес-логика приложения.
- **`alembic/`**: Миграции базы данных.
- **`tests/`**: Тесты API.

### 2. `bot` (Telegram Bot)
Интерфейс взаимодействия с пользователем через Telegram, реализованный на **python-telegram-bot**.
- **`main.py`**: Точка входа, содержит обработчики команд (`/start`), текстовых сообщений и callback-запросов. Реализует меню авторизации, просмотра плана лечения и экстренной помощи.

### 3. `sber_mock` (Mock Service)
Сервис-заглушка для имитации интеграции со СберЗдоровьем (или аналогичным сервисом).
- Реализован на **FastAPI**.
- Предоставляет тестовые эндпоинты для проверки интеграции без обращения к реальному внешнему API.

### 4. `db`
Конфигурационные файлы и скрипты для инициализации базы данных (PostgreSQL).

### 5. `pgadmin_config`
Конфигурация для PGAdmin - веб-интерфейса для управления PostgreSQL.

### 6. Корневые файлы
- **`docker-compose.yml`**: Описание оркестрации контейнеров (API, Bot, DB, PGAdmin, Mock).
- **`.env`**: Переменные окружения (необходимо создать на основе примера).
- **`requirements.txt`**: Зависимости проекта (могут быть разбиты по сервисам).
